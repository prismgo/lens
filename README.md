# Prismgo Lens

Prismgo Lens is a development-only helper module for PrismGo projects. It aligns the local developer experience with Laravel Boost while keeping MCP, Agent config, documentation search, browser logs, and skill auditing outside the production application build.

## Install

Install the Prismgo Lens CLI:

```bash
go install github.com/prismgo/lens/cmd/prismgolens@latest
```

Run from a PrismGo project root:

```bash
prismgolens install
```

The installer detects supported Agent config files, writes shared `.prismgo-lens.json` and local `.prismgo-lens.local.json`, creates `.ai/guidelines` and `.ai/skills`, merges MCP server config for detected Agents, and automatically adds the Lens binary directory to the user PATH when needed.

`.prismgo-lens.json` is the team-shareable config for enabled features, selected package modules, Agents, and MCP primitive filters. `.prismgo-lens.local.json` records machine-local state such as the absolute project root and detected Agents; it can be regenerated by `install` or `update` and should normally stay out of commits.

## Developing Lens From Source

When working inside the PrismGo repository before Lens is published, use the source checkout command:

```bash
go -C tools/prismgo-lens run . --project ../.. install
```

This is a development-only workflow. Published projects should use `prismgolens install`.

## Commands

```bash
prismgolens --help
prismgolens install
prismgolens install --interactive
prismgolens install --dry-run --no-interaction --agent codex --guidelines --mcp
prismgolens install --no-interaction --agent codex --package-module example.com/prismgo/pkg
prismgolens doctor
prismgolens update
prismgolens update --dry-run --ignore-skills
prismgolens mcp
prismgolens execute-tool application-info e30=
prismgolens browser-proxy --target http://127.0.0.1:8051 --listen 127.0.0.1:8052
prismgolens list-skills
prismgolens add-skill /path/to/local/skill
prismgolens add-skill owner/repo/path/to/skill
prismgolens add-skill owner/repo --list
prismgolens add-skill owner/repo --skill skill-name
prismgolens add-skill owner/repo --all
```

`install --dry-run` does not write files. When MCP is enabled, it prints the selected Agent MCP install plans, including strategy, config path, and stdio command.

`execute-tool` accepts base64-encoded JSON arguments. `e30=` is `{}`.

Project-specific documentation should live under the host project's `docs/` tree. Prismgo Lens does not require a root `prismgo/` source directory in published projects.

- `get-absolute-url`
- `get-env`

`tinker` is intentionally not implemented. Prismgo Lens does not run arbitrary Go code.

## Safety Boundaries

- The module lives under `tools/prismgo-lens` with its own `go.mod`.
- The module path is `github.com/prismgo/lens`.
- Production `go build .` does not import or compile this module.
- MCP calls go through the registered tool allowlist.
- MCP `tools/call` executes `execute-tool` in a subprocess with timeout and output limits.
- Generated MCP config runs `prismgolens --project . mcp` or an absolute `prismgolens` executable path, so tools resolve the host project instead of the Lens module.
- Codex TOML and JSON MCP config preserve unrelated servers and user keys while replacing only the `prismgo-lens` server. Corrupt JSON config, or non-object `mcpServers` / `servers`, is refused without rewriting the file.
- `database-connections` reads `config/database.go` plus `.env`, resolves env defaults, and redacts secrets.
- `application-info` includes a PrismGo roster with root module, Go directive, Go packages and replacements, frontend packages, and framework feature signals.
- `search-docs` supports `queries`, `packages`, and `token_limit` up to 1,000,000, and returns local source metadata such as package, language, and truncation status.
- `database-schema` has MySQL, PostgreSQL, SQLite, and unsupported-driver schema contracts. MySQL remains the default real connection path in this dev module; PostgreSQL and SQLite use the same output shape when a matching connection is available.
- `database-query` supports read-only MySQL access. Other drivers return an explicit unsupported error.
- `database-query` rejects write SQL, schema SQL, and multiple statements, enforces row/byte limits, and redacts secret-like columns.
- `get-env` and `get-config` redact or refuse secret-like keys.
- `list-routes` accepts real `route:list --json` output that uses `uri`, preserves original fields, and adds normalized `path` for Lens consumers.
- `list-console-commands` accepts real `list --format=json` output shaped as `{ "commands": [...] }`.
- `get-absolute-url` returns an explicit error when a route name cannot be resolved.
- `read-log-entries` can read `storage/logs/{channel}.log` or a relative `path` under `storage/logs`; JSON lines remain single entries and stack trace continuations are merged into the preceding entry.
- `browser-proxy` is a development-only reverse proxy. It injects the browser logger into successful, unencoded HTML responses; stream, binary, JSON, encoded, and non-2xx responses are passed through without buffering for injection.
- Browser log collection accepts `sendBeacon` `text/plain` payloads and formats console arguments into stable one-line entries before writing `storage/logs/browser.log`.
- Browser logger helpers are not registered in production routes.
- `.prismgo-lens.json` does not store the machine-local absolute project path. Lens writes `.prismgo-lens.local.json` for local state such as `project_root` and `detected_agents`.
- `install --interactive` runs the Boost-style local wizard. It lets the user choose Agents, guidelines, skills, MCP, browser logs, GitHub docs provider, and selected package modules. Non-TTY automation can keep using `--no-interaction` and flags.
- When `.prismgo-lens.json` enables `features.github_docs_provider`, `search-docs` merges local results with the optional GitHub JSON docs provider. Set `PRISMGO_LENS_GITHUB_DOCS_URL` to a prebuilt docs index endpoint; provider failures return local results plus a warning.
- HTTP MCP is not started or written by default. Agent adapters expose an HTTP MCP payload shape for future/manual use, while installed config continues to use local stdio.

## Skill Audit

`add-skill` installs a local skill directory or downloads a GitHub repository zip and discovers every `SKILL.md` under the requested path. Multiple remote skills require `--list`, `--skill NAME`, or `--all`; Prismgo Lens never silently installs the first candidate.

Local skill directories must contain `SKILL.md`; recursive self-copy sources are refused. GitHub downloads use a timeout and return explicit status errors for rate limits or failed responses.

High risk skills require `--force`. Critical risk skills require the explicit `--force --skip-audit` bypass after manual review. Installed skills are synced to detected Agent skill directories.

## Package Assets

`update --discover` lists AI assets from the current project's actual Go module dependencies. Dependencies may provide:

- `resources/prismgo-lens/guidelines/*.md`
- `resources/prismgo-lens/skills/{name}/SKILL.md`

Discovered assets are not enabled automatically. Add the module path to `.prismgo-lens.json` under `selected_package_modules`, or pass `install --package-module MODULE`; selected package guidelines are copied into `.ai/guidelines/packages/...`, selected package skills are audited and copied into `.ai/skills`, and the normal Agent sync writes them to configured Agent skill directories.
